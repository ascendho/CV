\section{项目经历}
\projentry{语义缓存增强的智能客服系统}{}{https://github.com/zhangsan/smart-cache}{https://zhangsan.dev/projects/smart-cache/demo}
\begin{itemize}
  \item \textbf{技术栈：}Python、FastAPI、LangGraph、LangChain、Redis
  \item \textbf{项目介绍：}面向电商客服场景的语义缓存增强系统，采用\projkeywordzh{混合检索}（向量召回 + 关键词匹配）与\projkeywordzh{两级缓存}架构：L1 通过规则匹配优先复用历史答案，L2 通过语义相似度召回候选；对于不确定的候选，由 \projkeyword{LLM Validator} 判断复用或回退到完整 RAG 流程。在 30 条评测请求上，缓存覆盖率达到 \projmetric{42.3\%}，端到端延迟降低 \projmetric{26.6\%}，费用节省 \projmetric{32.4\%}。
\end{itemize}

\projentry{多模态文档视觉检索与问答系统}{}{https://github.com/zhangsan/vision-doc}{https://zhangsan.dev/projects/vision-doc/demo}
\begin{itemize}
  \item \textbf{技术栈：}Python、FastAPI、ColPali、Qdrant、pdf2image、Docker
  \item \textbf{项目介绍：}聚焦图文混排、复杂版式文档的检索需求，构建多模态页级证据检索与问答系统。上传阶段将 PDF、PPTX 统一转为页面图像，使用 ColPali 生成多向量视觉特征并写入 Qdrant 建立索引。查询阶段通过\projkeywordzh{复合问题拆分}与\projkeywordzh{多轮对话上下文改写}，结合 \projkeyword{MUVERA Prefetch + ColPali MaxSim} 两阶段精排，实现高精度页面定位。在 20 题测评中，Hit Rate@3 达到 \projmetric{90\%}，MRR 达到 \projmetric{0.77}。
\end{itemize}

\projentry{基于 SFT + GRPO 的推理能力两阶段微调}{}{https://github.com/zhangsan/reasoning-finetune}{}
\begin{itemize}
  \item \textbf{技术栈：}Python、Predibase、LoRA、SFT、GRPO
  \item \textbf{项目介绍：}围绕结构化推理任务，基于 \textit{Qwen 2.5 7B Instruct} 搭建两阶段微调流水线：先通过 SFT 对齐结构化输出格式与基础推理能力，再通过 GRPO 引入\projkeywordzh{格式校验}、\projkeywordzh{历史一致性}、\projkeywordzh{信息增益}三类奖励函数强化策略学习。在 10 局基准测试中取得 \projmetric{7/10} 通关率，表现\projkeywordzh{超过} \textit{GPT-4o-mini}（1/10），\projkeywordzh{逼近} \textit{Claude 3.5 Sonnet}（8/10）。
\end{itemize}
